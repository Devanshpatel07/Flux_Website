## Phase 1: Setup MongoDB Atlas (Cloud Database)
Vercel and Render run in the cloud, so they cannot connect to `localhost:27017`. You need a cloud MongoDB database.

1. Go to [MongoDB Atlas](https://www.mongodb.com/products/platform/atlas-database) and sign in/register for a free account.
2. Click **Create a Cluster** (choose the **M0 Shared Free Tier**).
3. **Security Access Setup**:
   - **Database User**: Create a username and password (write these down).
   - **IP Access List**: Select **Allow Access from Anywhere (`0.0.0.0/0`)** since Render/Vercel server IPs rotate.
4. Go to **Database > Connect > Drivers**. 
5. Copy the connection string. It will look like this:
   ```text
   mongodb+srv://<username>:<password>@cluster0.xxxx.mongodb.net/?retryWrites=true&w=majority&appName=Cluster0
   ```
6. Replace `<username>` and `<password>` with the database credentials you created in step 3. Make sure to choose a database name (e.g. `flux` instead of `/` before `?`). Keep this connection string ready.

## Phase 2: Deploy Backend to Render (supports WebSockets/Chat)
We choose **Render** for the backend over Vercel because Vercel utilizes stateless Serverless Functions which **do not** support the persistent WebSocket connections (Socket.io) used by the Flux Group Chat. Render supports full, long-lived State Web Services.

1. Go to [Render.com](https://render.com/) and log in using your GitHub account.
2. Click **New + > Web Service**.
3. Connect your **`Flux_Website`** GitHub repository.
4. Set the following configuration values:
   - **Name**: `flux-backend`
   - **Environment**: `Node`
   - **Root Directory**: `FLUX-BACKEND`
   - **Build Command**: `npm install`
   - **Start Command**: `node server.js`
5. Scroll down to **Environment Variables** and add the following keys:
   - `MONGO_URI` = *(Your MongoDB Atlas connection string from Phase 1)*
   - `JWT_SECRET` = `supersecretfluxwavejwttokenkey12345` (or any secure random string)
   - `NODE_ENV` = `production`
6. Click **Create Web Service**.
7. Wait 2-3 minutes for the build to finish. Once completed, copy the live URL generated by Render. It will look like: 
   ```text
   https://flux-backend-xxxx.onrender.com
   ```

## Phase 3: Deploy Frontend to Vercel
Vercel is ideal for high-performance frontend hosting.

1. Go to the [Vercel Dashboard](https://vercel.com/) and log in using GitHub.
2. Click **Add New > Project** and import the `Flux_Website` repository.
3. Configure the Project settings:
   - **Framework Preset**: Select **`Vite`** (auto-detected).
   - **Root Directory**: Click Edit, select **`FLUX-FRONTEND`**, and click Continue.
4. Under **Build and Output Settings** (keep defaults):
   - Build Command: `npm run build`
   - Output Directory: `dist`
5. Under **Environment Variables**, add the single environment key:
   - **Key**: `VITE_BACKEND_URL`
   - **Value**: `https://flux-backend-xxxx.onrender.com/api` *(Paste the URL of your deployed Render service from Phase 2, making sure to append `/api` at the end)*
6. Click **Deploy**.
7. Once deployed, Vercel will give you a live production URL (e.g. `https://flux-website.vercel.app`).

## ⚡ Synchronizing the Frontend and Backend CORS
To permit your new frontend URL to communicate with the Render API, make sure your frontend address is listed in the server permitted origins.
*(Note: Our latest commit automatically allows any origins ending in `.vercel.app`, so your frontend should connect without any extra changes!)*
